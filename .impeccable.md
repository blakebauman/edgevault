# Design Personality — EdgeVault

Captured from the designer's three-question interview on 2026-06-05, during the
`/stardust:brand` stage of an end-to-end pipeline run.

## References the designer likes

- **Tailscale / Fly.io** — plainspoken engineering voice, technical editorial
  register, a little wit. Copy makes concrete claims and trusts the reader.
- **Swiss / international style** — grid discipline, typographic clarity, no
  ornament. Hierarchy does the work; decoration doesn't.
- **infisical.com** — named explicitly by the designer. A direct category
  neighbor (secrets management). Treat as a *taste* anchor, not an extraction
  source: developer-first presentation, real product surfaces, dark technical
  ground.

## Pet peeves (all four confirmed — treat as hard anti-rules)

1. **Gradient-blob heroes** — no purple-glow mesh gradients, no floating 3D
   shapes, no generic SaaS hero decoration. Ever.
2. **Marketing fluff copy** — no "supercharge your workflow". Every claim must
   be concrete and checkable (a latency number, a consistency guarantee, an
   architecture fact).
3. **Inter-everywhere sameness** — do not default to the Inter + pill-button +
   dark-hero template. Font choices must be a deliberate deck pick; Inter is
   effectively banned as the headline/identity face.
4. **Fake dashboards** — no blurry invented product screenshots. If a product
   surface is shown, it is a *real artifact*: actual config syntax, actual CLI
   output, actual API responses, actual SDK code.

## One rule the designer wants broken

**No hero illustration.** The hero leads with real artifacts — an actual
config file, actual CLI output, an actual `curl` against the delivery plane —
instead of decoration. The product's own material is the visual.

## How downstream skills apply this

- `briefings`: copy must pass peeve #2 — concrete claims only. Hero copy
  pairs with a real artifact, not an illustration brief.
- `wireframes`: hero section is artifact-led (`data-intent` should say so).
- `prototype`: enforce peeves #1, #3, #4 as render-blocking rules. The
  rule-break (artifact-as-hero) is a required move, not an option.
